dirigir las preguntas al WhatsApp solicitado y revisar referencias de imagenes.

  iniciar la aplicacion en Railway.

CAMBIOS REALIZADOS
1. Se creo app.py como punto de entrada Flask.
2. Se agrego la ruta principal / para servir el catalogo.
3. Se agrego la ruta /health para que Railway pueda validar el servicio.
4. Se organizo el proyecto con la estructura estandar de Flask:
   - templates/index.html
   - static/css/style.css
   - static/js/script.js
   - static/images/
5. Se agrego requirements.txt con Flask y gunicorn.
6. Se agregaron Procfile y railway.json para iniciar gunicorn app:app.
7. Se agrego .gitignore para excluir entornos virtuales, cache y .env.
8. Se actualizaron las rutas del HTML para usar url_for de Flask.
9. Se actualizaron las rutas de imagenes generadas desde JavaScript.
10. Se retiro del catalogo la informacion de las tres tarjetas con fotografias
    inexistentes para evitar imagenes rotas en produccion.
11. Se configuro el WhatsApp solicitado con formato internacional:
    524612101815. Los botones generales y los botones de cada bolsa usan este
    numero mediante enlaces wa.me.
12. Se agregaron encabezados HTTP basicos de seguridad:
    Content-Security-Policy, Referrer-Policy, X-Content-Type-Options,
    X-Frame-Options y Permissions-Policy.
13. Se agrego README.md con instrucciones para ejecutar localmente y desplegar
    en Railway.
